# Amazon S3 Provider

Uses boto3 to upload files to an S3 bucket.

## Arguments
- `bucket` – name of the target S3 bucket.
- `prefix` – optional prefix inside the bucket.

## AWS Credentials

Credentials can be provided in the provider section or via a YAML config file:

```ini
[provider-cloud]
type: s3
bucket: mybucket
region: us-east-1
access key id: AKIAIOSFODNN7EXAMPLE
secret access key: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
```

Alternatively use a named profile:
```ini
profile: myprofile
```

Or point to a YAML config file:
```ini
aws config: /path/to/aws.yaml
```

The YAML file may contain: `region`, `access_key_id`, `secret_access_key`,
`session_token`, `profile`, `endpoint_url`.

## Pros
- Objects can be stored in immutable storage classes (e.g. Glacier or Glacier Deep Archive) which protects against ransomware.
- Highly durable and available.

## Cons
- Transfer costs may apply.
